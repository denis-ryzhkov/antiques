алгоритм распознавания образов (C) преподаватель Образцов В.А.,
после осмысления (C) студент Рыжков Д.В.

задача распознавания образов состоит в автоматизации процедуры
выбора класса для объекта.

сперва система находится в режиме обучения, затем в режиме распознавания.

для обучения системе даётся множество объектов, уже разбитых на классы.

это множество разделяется на две непустые непересекающиеся части -
параметрическую и обучающую, так чтобы каждый класс оказался разделён
аналогичным образом.
      
выбирается вещественная функция близости объектов, например обычная метрика.
и вычисляется матрица близости обучающих объектов к параметрическим объектам.

для каждого класса выбирается вещественная функция
близости объекта к данному классу, зависящая от функции близости объектов,
например взвешенная сумма функций близости данного объекта к каждому
параметрическому объекту данного класса.

для каждого класса выбирается решающее правило - булева функция от объекта,
зависящая от функций близости к классам, например обращающаяся в истину
лишь для класса, к которому ближе всего данный объект.

начало цикла

для каждого обучающего объекта и каждого параметрического класса
вычисляем вещественную функцию близости объекта к классу на основе
матрицы близости объектов. получаем матрицу из вещественных чисел.

для каждого обучающего объекта вычисляем решающее правило,
что отнесёт данный объект к некоторому параметрическому классу.

если этот параметрический класс соответствует обучающему классу данного объекта,
то данный объект распознан правильно.

вычисляется функционал качества, равный отношению правильно распознанных
обучающих объектов ко всем обучающим объектам.

если качество распознавания устраивает, то выходим из цикла.
иначе корректируем функции близости объектов к классам и/или решающее правило.

конец цикла.

в режиме распознавания используются функции полученные в режиме обучения.

для каждого ещё не классифицированного объекта вычисляются:
функция близости объектов, функция близости объектов к классам и
решающее правило, которое и определяет класс объекта.

конец алгоритма.
